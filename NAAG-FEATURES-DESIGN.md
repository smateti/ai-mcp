| Field | Type | Description |
|-------|------|-------------|
| id | BIGINT | Auto-increment primary key |
| timestamp | TIMESTAMP | When the action occurred |

| user_role | VARCHAR(50) | ADMIN or USER |
| session_id | VARCHAR(100) | Session identifier |
| service_name | VARCHAR(100) | Which NAAG service |
| action_type | VARCHAR(50) | CHAT, TOOL_CALL, RAG_QUERY, etc. |
| question | TEXT | User's input |
| answer | TEXT | System's response |
| tool_name | VARCHAR(100) | Tool executed (if applicable) |
| tool_parameters | TEXT | JSON parameters |
| tool_result | TEXT | JSON result |
| status | VARCHAR(20) | SUCCESS, FAILED, PARTIAL, TIMEOUT |
| duration_ms | BIGINT | Processing time |
| confidence | DECIMAL(3,2) | LLM confidence score (0.00-1.00) |

## 7. Enhanced Document Upload with Q&A Preview

### Overview

The enhanced document upload feature provides a robust workflow for validating document quality before adding it to the RAG knowledge base. The system generates Q&A pairs from the document, tests retrieval against a temporary collection, and allows administrators to review and approve before finalizing.

```yaml
  rag:
    qa-generation:
      fine-grain-count: 5        # Detailed Q&A pairs
      summary-count: 3           # High-level Q&A pairs
```

### SSE Event Types

| Event Type | Description |
|------------|-------------|
| `processing` | Progress update during async processing |
| `completed` | Document ready for review |
| `error` | Processing failed |

### API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| /api/documents/upload | POST | Upload document for preview |
| /api/documents/uploads | GET | List all uploads |
| /api/documents/uploads/{id} | GET | Get upload details + Q&A |
| /api/documents/uploads/{id}/approve | POST | Approve and move to RAG |
| /api/documents/uploads/{id}/retry | POST | Reprocess document |
| /api/documents/uploads/{id} | DELETE | Delete upload |
| /api/documents/uploads/{id}/events | GET | SSE for specific upload |
| /api/documents/events | GET | Global SSE notifications |

### application.yml Template

```yaml
spring:
  application:
    name: naag-service-name

  # H2 Database (Development)
  datasource:
    url: jdbc:h2:file:./data/naag-db;AUTO_SERVER=TRUE
    driver-class-name: org.h2.Driver

  # JCache/EhCache
  cache:
    type: jcache
    jcache:
      config: classpath:ehcache.xml

naag:
  # Audit
  audit:
    enabled: true
    storage: h2  # h2, elasticsearch, both

  # Cache
  cache:
    enabled: true
    default-ttl-seconds: 60

  # Q&A Generation (for naag-rag-service)
  rag:
    qa-generation:
      fine-grain-count: 5
      summary-count: 3
      validation-similarity-threshold: 0.7

# Actuator
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus,caches
```

## 9. Summary

### Platform Features

| Feature | Description | Technology |
|---------|-------------|------------|
| **Audit Logging** | All user interactions logged | H2/Elasticsearch |
| **User Context** | User ID, role tracking | HTTP Headers |
| **User Roles** | ADMIN (full), USER (chat) | Prepared for security |
| **Caching** | Configurable TTL caching | EhCache 3 / JCache |
| **Chat UI** | User chat with tools & RAG | Thymeleaf + Bootstrap |
| **Admin UI** | Category/Tool/Doc management | Thymeleaf + Bootstrap |
| **Document Preview** | Q&A validation before RAG | Async + SSE |
| **Service Health** | Dashboard monitoring | Actuator endpoints |
